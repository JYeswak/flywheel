Local preflight

GitHub Actions should be final approval, not the first place you find basic breakage.

I favor local containers, dry-runs, doctor receipts, and screenshot checks before spending hosted CI minutes.

Support tiers

Each lane has to earn its label.

The developer path has to work inside and outside NTM. Reduced local mode is required, and 4 named agent lanes carry isolated runtime receipts before I call them supported.

RequiredReduced local mode

Proven fallback path for preflight, init, doctor, tick, dispatch simulation, closeout, and next action without private substrate.

ProvenClaude Code

Isolated runtime receipt proves the lane with a live adapter response and clean private-state scan.

ProvenCodex CLI

Isolated runtime receipt proves the lane using an explicit operator-provided Codex auth home while target repo and HOME stay isolated.

ProvenGemini CLI

Isolated runtime receipt proves the lane with live adapter response and the same required journey stages.

ProvenOpenClaw

Isolated runtime receipt proves behavior through a disposable isolated OpenClaw agent.

Cost discipline

Fast systems should not burn expensive context to prove routine work.

The same principle that makes Flywheel safer for SMB workflows makes it cheaper to operate: grep before fetching, batch routine checks, route the right model to the right task, and save hosted CI for final confirmation.
