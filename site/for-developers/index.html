The foundation

Jeff Emanuel built the substrate I stand on.

I did not build NTM, Agent Mail, beads, or CASS. Jeff Emanuel, also known as Dicklesworthstone, did. I use his work because it solves the local-agent problems that matter before any higher layer is useful.

Jeff EmanuelNTM

Named Tmux Manager gives agent panes stable names, session shape, and a way to route work without guessing which terminal is which.

Jeff EmanuelAgent Mail

Agent Mail gives agents durable messages, acknowledgements, and thread history so cross-agent work does not vanish into scrollback.

Jeff Emanuelbeads

beads gives the repo a local issue graph: work can be opened, blocked, linked, closed, and inspected without a hosted tracker.

Jeff EmanuelCASS

CASS-style memory keeps useful context available without forcing every agent to rediscover the same facts every session.

What I built on top

Flywheel and SkillOS turn the substrate into a working loop.

Flywheel is my repo-local operating loop. SkillOS is the capability layer beside it. Together they make agent work easier to inspect, reuse, and run across Claude Code, Codex CLI, Gemini CLI, and OpenClaw.

FlywheelRepo-local loop

Mission, goal, state, doctor checks, daily reports, local preflight, and closeout receipts live with the repo.

SkillOSCapability control

Skills become explicit capabilities with validators, adoption proof, and pack-level status instead of ambient prompt memory.

ResearchKnowledge packs

External changes can become testable, reproducible packs that agents can load instead of re-learning the same update.

RoutingRight model, right task

Planning, implementation, cleanup, and local checks do not need the same model or the same context every time.

ProofBlocked beats guessed

When a claim is not proven, the system says so. Red evidence is useful because it keeps the next step honest.

How it works

The loop is simple enough to study and strict enough to keep.

A repo starts with a mission. Agents take bounded work. Each run leaves receipts, tests, and lessons. Useful lessons move into skills or packs. The next repo starts with more of the map already drawn.

01Parallel agents

Claude Code, Codex CLI, Gemini CLI, and OpenClaw can each work from the same repo-local state without assuming one private runtime.

02Shared feedback

Daily reports, closeout receipts, and beads turn agent work into inspectable history instead of one long chat transcript.

03Reusable knowledge

When internet research or repo work proves useful, SkillOS can turn it into a pack with validators and status.

04Local gates first

Preflight, doctor, visual checks, and local containers catch basic breakage before GitHub Actions becomes final approval.

Take it

There are two honest doors.

You can use Jeff Emanuel's open-source tools directly. You can also clone Flywheel and SkillOS, run the installer locally, and keep working without me. The install path is scoped to ~/.flywheel/engine; dry-run is available before files are copied.

LOCAL INSTALLDRY-RUN FIRST
git clone https://github.com/JYeswak/flywheel.git
git clone https://github.com/JYeswak/zeststream-skillos.git
cd flywheel
bash install.sh --dry-run --json
bash install.sh --json
