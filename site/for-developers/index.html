The foundation

Jeffrey Emanuel built the substrate I stand on.

I did not build NTM, Agent Mail, beads, CASS, JSM, dcg, ubs, caam, or FrankenSQLite. Jeffrey Emanuel, also known as Dicklesworthstone, did. I use his work where it helps, and I keep Flywheel and SkillOS honest about what also works without it.

Jeffrey EmanuelNTM

Named Tmux Manager gives agent panes stable names, session shape, and work commands without guessing which terminal is which.

Jeffrey EmanuelAgent Mail

Agent Mail gives agents durable messages, acknowledgements, searchable threads, and advisory file leases.

Jeffrey Emanuelbeads

beads gives the repo a local issue graph: work can be opened, blocked, linked, closed, and inspected without a hosted tracker.

Jeffrey EmanuelCASS

CASS indexes local coding-agent session history so agents can search prior work without loading whole transcripts.

Jeffrey EmanuelJSM

Jeffrey Skills Manager is the paid skill feed I treat as upstream-owned: SkillOS tracks adoption and safety without rewriting JSM-managed content.

Jeffrey Emanueldcg

Destructive Command Guard blocks risky shell and git commands before a coding agent turns speed into damage.

Jeffrey Emanuelubs

Ultimate Bug Scanner runs pattern checks before changes harden into repo history.

Jeffrey Emanuelcaam

Coding Agent Account Manager isolates and switches agent auth profiles so parallel work does not share one fragile login state.

Jeffrey Emanuelfrankensqlite

FrankenSQLite is the database research line I study for durable local ledgers and concurrent-write ideas.

What I built on top

Flywheel and SkillOS can stand alone or sit on Jeffrey Emanuel's substrate.

Flywheel is my repo-local operating loop. SkillOS is the capability layer beside it. The reduced public path does not require NTM, Agent Mail, or a private skill library; the fuller path uses those tools when you install them.

FlywheelRepo-local loop

Mission, goal, state, doctor checks, daily reports, local preflight, and closeout receipts live with the repo.

SkillOSCapability control

Skills become explicit capabilities with validators, adoption proof, and pack-level status instead of ambient prompt memory.

ResearchKnowledge packs

External changes can become testable, reproducible packs that agents can load instead of re-learning the same update.

RoutingRight model, right task

Planning, implementation, cleanup, and local checks do not need the same model or the same context every time.

ProofBlocked beats guessed

When a claim is not proven, the system says so. Red evidence is useful because it keeps the next step honest.

How it works

The loop is simple enough to study and strict enough to keep.

A repo starts with a mission. Agents take bounded work. Each run leaves receipts, tests, and lessons. Useful lessons move into skills or packs. The next repo starts with more of the map already drawn.

01Parallel agents

Claude Code, Codex CLI, Gemini CLI, and OpenClaw can each work from the same repo-local state without assuming one private runtime.

02Shared feedback

Daily reports, closeout receipts, and beads turn agent work into inspectable history instead of one long chat transcript.

03Reusable knowledge

When internet research or repo work proves useful, SkillOS can turn it into a pack with validators and status.

04Local gates first

Preflight, doctor, visual checks, and local containers catch basic breakage before GitHub Actions becomes final approval.

My systemBrand voice — copy that cannot quietly lie

AI marketing copy can sound perfectly on-brand and still invent the facts. This system checks every factual claim against a grounded claim bank at the moment of writing, and rejects the copy when a claim has no receipt.

Take it

There are two honest doors.

You can use Jeffrey Emanuel's open-source tools directly. You can also clone Flywheel and SkillOS, run the installer locally, and keep working without me. The reduced path is standalone; the fuller path uses Jeffrey Emanuel's substrate when it is installed. The install path is scoped to ~/.flywheel/engine; dry-run is available before files are copied. The hosted install endpoint is for checksum parity, not a curl-only install path.

LOCAL INSTALLDRY-RUN FIRST
git clone https://github.com/JYeswak/flywheel.git
git clone https://github.com/JYeswak/SkillOS.git
cd flywheel
bash install.sh --dry-run --json
bash install.sh --json
