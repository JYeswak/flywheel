# Cluster Maintainer Pattern

Imported from `.flywheel/doctrine/cluster-maintainer-pattern.md` (ti46c, Phase
2 of 38u3d). Stub summary only — see the upstream doctrine doc for canonical
text.

## Rule (one line)

When N>=3 sister beads land the same shape, promote one of them to
cluster-maintainer status: it owns the recipe, the test-receiver, and the
forward-link to future sisters.

## When to apply

- 3 beads ship the same fix shape within a session
- A worker discovers a recurring pattern via convergent evolution
- A skill repeatedly produces look-alike artifacts

## Why it exists

Without a cluster-maintainer, the recipe drifts: each sister re-derives the
shape, the test-receiver fragments, and the forward-link decays. Promotion
freezes the recipe and assigns ownership.
