# Hook Trauma Class Promotion Discipline

When hooks emit the same trauma class repeatedly, the class must not remain a
private log pattern. It needs a durable routing surface: memory, doctrine,
incident, probe, dashboard, or canonical rule. A hook that catches the same
failure hundreds of times without promotion is not a warning system; it is a
ledger of ignored pain.

- The emitted trauma class that proves the rule is still failing in practice.

## Operating Rule

If a hook-authored trauma class reaches promotion threshold, do one of these in
the same slice:

- Wire a structural receiver that consumes the class.
- Promote a doctrine or incident page that names the class exactly.
- Create a bead for the missing receiver with an acceptance test.
- Record an explicit skip receipt that says why the class is not actionable.

Do not leave a high-N hook class as "observed." Observed but unwired is the
failure class.
