## The ZestStream Agentic Stack

The methodology is the **Agentic Coding Flywheel**: a 9-petal loop (Intent →
Plan → Multi-model competition → Synthesis → Review → Beads → Triage → Swarm
→ Learn/Reuse) that runs across `tmux` panes via Jeff Emanuel's
[NTM](https://github.com/Dicklesworthstone/ntm) /
[Beads Rust](https://github.com/Dicklesworthstone/beads_rust) /
[Agent Mail](https://github.com/Dicklesworthstone/mcp_agent_mail) /
[DCG](https://github.com/Dicklesworthstone/destructive_command_guard) /
[CASS](https://github.com/Dicklesworthstone/coding_agent_session_search) /
[JSM](https://github.com/Dicklesworthstone/meta_skill) substrate.

What ZestStream adds is the **skill-and-doctrine layer on top** — the
operating discipline that turns a bunch of AI tools into a self-sustaining
fleet that runs a consulting business without me touching every keystroke.

## The Agentic Coding Flywheel

The doctrine ZestStream is built on. Borrowed in spirit from Jeff Emanuel's
[Agentic Coding Flywheel](https://agent-flywheel.com) and extended with the
**Three Reasoning Spaces** model (PLAN / BEAD / CODE) and **Skill OS**.

```
   1. INTENT  →  2. FIRST PLAN  →  3. MULTI-MODEL  →  4. SYNTHESIS
        ↑                            COMPETITION              │
        │                                                     ↓
   9. LEARN/REUSE                              5. REVIEW/TEST/HARDEN
        ↑                                                     │
        │                                                     ↓
   8. SWARM EXECUTION  ←  7. TRIAGE/DISPATCH  ←  6. CONVERT TO BEADS
```

Two practitioner rules that fall out of this:

1. **Think globally in plans, operationally in beads, locally in code.**
   Mistake cost climbs ~5× from plan → bead and ~25× from plan → code, so
   the highest-leverage hours are spent in plan-space, not editor-space.
2. **AI proposes, Joshua disposes.** Multi-model triangulation produces
   candidates; taste is human, and the operator owns the disposition.

Detailed doctrine: [flywheel/.flywheel/AGENTS-CANONICAL.md](https://github.com/jyeswak/flywheel/blob/master/.flywheel/AGENTS-CANONICAL.md).

## Philosophy

> [!IMPORTANT]
> The bottleneck isn't writing code — it's the discipline of running a fleet
> of agents without lying to yourself about what they shipped. Build the
> tooling that **forces honest receipts** at every loop boundary and the
> compounding works in your favor. Build it sloppy and it works against you.

Recurring threads:
- **Receipts over promises.** Every callback is a structured envelope;
  every bead closes with verifiable evidence; every doctrine rule cites the
  failure mode that caused it
- **Most intelligence in plan-space.** Refactoring is ~10× cheaper before
  shipping and ~25× cheaper before users. The flywheel optimizes for
  catching expensive mistakes in plan, not in production
- **Self-sustaining over heroic.** SpaceX/Tesla shape: architecture-health
  over individual heroics. The fleet must run when Joshua is not at the
  keyboard
