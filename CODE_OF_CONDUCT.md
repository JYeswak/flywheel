# Code Of Conduct

Flywheel contributors are expected to keep technical disagreement direct,
specific, and evidence-based.

## Standards

Acceptable behavior includes:

- focusing criticism on code, docs, tests, receipts, and user impact;
- naming risks and tradeoffs clearly;
- preserving private data, credentials, and client confidentiality;
- helping contributors reproduce failures with commands or evidence paths;
- respecting maintainers' publication, safety, and scope decisions.

Unacceptable behavior includes:

- harassment, threats, or personal attacks;
- publishing private information, credentials, or non-public client details;
- knowingly bypassing safety gates or encouraging others to do so;
- misrepresenting unsupported integrations as tested support;
- repeated off-topic disruption after maintainers ask for a thread to stop.

## Enforcement

Report conduct issues to `joshua@zeststream.ai`. Maintainers may edit, hide, or
remove comments; close issues; block users; or decline contributions that violate
this code or the repository's safety rules.

Good faith security research and direct technical critique are welcome. The line
is whether the behavior helps the project stay truthful, inspectable, and safe.
